Formalización y Pronto Pago

Rutas prioritarias para aplicación de descuentos y formalización de matrícula.

Formalización de Matrícula

Plantilla de Formalización

Ruta en Class:

Módulo de Cajas Pago facturación Facturación Colocar Cédula Clic en Buscar (x2)

¿Cuándo se usa? Cuando el estudiante quiere incluir la matrícula dentro de la financiación o cuando quiere financiar y la matrícula le sale gratis.

Paso a paso en la tabla de materias:

  1. En la tabla de la derecha verás las columnas: Elim | Finan | Efec. | Desc.
  2. La matrícula siempre saldrá en negro (Efec) por defecto.
  3. Debes dar clic en Finan para que pase a color Rojo.
    (Esto la incluye en la financiación, incluso si sale gratis).
  4. En la parte de arriba selecciona Cobros ➔ Aceptar ➔ Aceptar.

¡Importante! Al finalizar, se genera factura y pagaré. Siempre debes enviar el informe de matrícula y el pagaré.

Aplicar Descuento Pronto Pago

¿Cuándo se usa? Si el estudiante va a pagar de contado (matrícula y materias). Luego el estudiante paga por U virtual, BN o BCR.

Misma ruta inicial, pero cambian las acciones en la tabla:

  1. Asegúrate de dar clic en Efec para que todo quede en color Negro.
  2. Da clic en el botón que dice Desc.
  3. De la tabla desplegable, selecciona "Descuento pronto pago y adicional".

Esta acción de aplicar el descuento se debe repetir con todas las materias y la matrícula, aun cuando la matrícula salga gratis.

Estandarización de Documentos

Es importante recordar que estas constancias pueden ser presentadas ante otras instituciones, por lo que deben guardarse de forma correcta y estandarizada. La U nos comparte el formato establecido para garantizar uniformidad y evitar inconvenientes.

Proceso correcto de guardado:

Número de cédula – Nombre completo – Abreviatura correcta

1. Docs Académicos

  • HN - Histórico de Notas
  • C - Estudiante Activo
  • IM - Informe Matrícula
  • CM - Constancia Matrícula

2. Docs Financieros

  • P - Proforma
  • CND - No Deuda
  • CD - Deuda

3. Docs Adicionales

  • D - Devolución

Estudiante Activo

Académico ➔ Certificaciones ➔ Cédula ➔ Carrera Activa ➔ Lista reportes ➔ Constancia ➔ Atención al Cliente ➔ Generar
  • Aplica solo si cursa el cuatrimestre actual (no inactivos).
  • Revisar y modificar la ortografía antes de enviar.
  • Fechas opcionales a agregar: "Iniciando lecciones del [XXX] cuatrimestre el [XX] de [XXXXXX] y culminando el [XX] de [XXXXXX] del [XXXX]."

Constancia de Interese

Certificaciones ➔ Cédula ➔ Carrera Activa ➔ Lista de reportes ➔ Interese
  • Sirve para saber cuántas materias ha cursado (virtual/presencial).
  • Nota: No aparecen materias actuales. Contarlas validando el historial.
  • NO COMPARTIR CON EL ESTUDIANTE (Solo uso interno).

Saldo Pendiente

Financiamiento ➔ Constancia saldo ➔ Constancia general ➔ Cédula ➔ Imprimir / Generar
  • Se debe modificar el saldo sumando el interés de mora (el sistema arroja monto sin intereses).

Validación Saldo (Interno)

Mód. Cajas ➔ Pago financiamiento ➔ Cliente: Cédula ➔ Buscar ➔ Listado (Saldo principal) ➔ Nº Operación
  • Se usa cuando U Virtual tiene el cronómetro y no permite validar el saldo.

Factura Electrónica

Fact. Electrónica ➔ Procesos ➔ Consulta ➔ Nº CIF/Cédula ➔ Cuatrimestre ➔ Reimprimir (PDF) ➔ Lápiz (XML / Acuse)
  • Se envía directo a solicitud del estudiante (sin hacer caso).
  • Se envían 3 docs: PDF, XML y Acuse XML.
  • Sale sin intereses. Se genera factura adicional por intereses.

Comprobante / Letra de Cambio

Mód. Cajas ➔ Reportes ➔ Estados pagos ➔ Cédula ➔ Fechas ➔ Reimpresión / Letra
  • Letra de cambio: Enviar siempre que se pague matrícula por U Virtual, junto con informe de matrícula.
